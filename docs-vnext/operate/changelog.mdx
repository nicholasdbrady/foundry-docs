This page tracks significant changes to the Microsoft Foundry documentation site. For platform release notes, see the [Azure updates page](https://azure.microsoft.com/updates/).

<Update label="2026-06-16" description="Major content expansion: 280 new pages across all documentation sections">

## Added

- **[Agents — development](/agents/development/overview)** — 26 new pages covering hosted agents, capability hosts, agent lifecycle, deployment, publish flows, prompt optimization, and agent tracing (concept, framework, and setup)
- **[Agents — tools](/agents/tools/web-overview)** — 34 new pages covering computer use, agent-to-agent, browser automation, MCP server, code interpreter, custom code interpreter, Bing tools, AI Search, file search, function calling, image generation, Fabric, and voice agent integration
- **[Models — capabilities](/models/capabilities)** — 38 new pages covering chat completions, audio, batch, embeddings, image generation, reasoning, predicted outputs, JSON mode, and speech
- **[Models — catalog](/models/catalog)** — 25 new pages with per-model deployment guides and benchmarks
- **[Models — fine-tuning](/models/fine-tuning)** — 10 new pages covering fine-tuning workflows and CLI
- **[Observability](/observability)** — 24 new pages covering evaluation datasets, synthetic data generation, rubric evaluators, trace data, end-user feedback logging, external agent registration, and troubleshooting
- **[Security](/security)** — 20 new pages covering RBAC, managed virtual networks, private link, encryption keys, custom policy definitions, and high availability
- **[Setup](/setup)** — 20 new pages covering resource creation (portal, Terraform, templates), migration, quotas, standard agent setup, and virtual networks
- **[API & SDK reference](/api-sdk/sdk-overview)** — 17 new pages covering SDK quickstarts and reference
- **[Guardrails](/guardrails/guardrails-overview)** — 15 new pages covering content filtering, prompt shields, protected material detection, custom categories, and safety system message templates
- **[Developer experience](/developer-experience/get-started)** — 11 new pages covering VS Code workflows (pro-code and low-code), AI templates, CLI/SDK setup, and security best practices
- **[Manage](/manage/overview)** — 9 new pages covering cost management, governance, and resource operations
- **[Responsible AI](/responsible-ai/overview)** — 8 new pages covering content safety, fairness, and responsible use guidelines
- **[Operate](/operate/whats-new-foundry)** — 8 new pages covering known issues, status dashboard, model lifecycle/retirements, region support, and accessibility
- **[Get started](/get-started/quickstart-create-foundry-resources)** — 6 new pages covering quickstart resource creation, developer journey, and AI tool connections
- **[Best practices](/best-practices)** — 5 new pages covering prompt engineering, system messages, and model-specific guidance
- **[Overview](/overview/what-is-foundry)** — 3 new pages: what is Foundry, developer guide, and classic-to-new migration reference

</Update>

<Update label="March 2026" description="AI-native features and developer experience improvements">

## Added

- **Contextual menu** on all pages with options to copy as Markdown, view as Markdown, and open in AI tools (ChatGPT, Claude, Perplexity, Grok, GitHub Copilot)
- **MCP server connections** for VS Code and VS Code Insiders directly from the contextual menu
- **[Connect AI tools](/get-started/connect-ai-tools)** guide with step-by-step instructions for Claude Desktop, ChatGPT, GitHub Copilot, Cursor, Perplexity, and Grok
- **[AI ingestion formats](/get-started/ai-ingestion)** guide explaining Markdown pages, llms.txt, llms-full.txt, skill.md, and MCP server
- **AI assistant configuration** with Microsoft Foundry-specific persona, terminology, and product context
- **Documentation agent rules** (AGENTS.md) for consistent automated documentation contributions
- **Page feedback** with thumbs up/down rating, edit suggestions, and issue creation
- **This changelog** for tracking documentation updates

## Improved

- Documentation site now serves all pages as Markdown to AI agents for reduced token usage
- MCP server provides live search across all indexed documentation pages
- skill.md provides structured product context for AI agent skill registration

</Update>
